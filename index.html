POC de Factoring

Interfaz de prueba para los dos flujos principales del POC: registrar/publicar una factura y comprar una participacion sobre una factura disponible.

Critical Path

Alta y validacion de factura

Prueba el flujo `POST /facturas`, incluyendo validacion, rechazo y publicacion.

Happy Path

Compra de una factura

Prueba `POST /facturas/{id}/comprar`, el saldo del inversionista y el tracking del proceso.

Registrar factura

Usa este tab para probar validaciones exitosas o fallidas del flujo de alta.

POST /facturas
Pruebas rapidas:
`serie` iniciando con `ERR` hace que la API responda con falla tecnica del mock SUNAT.
`correlativo` terminando en `999` se puede ingresar, pero al enviar la factura la API la marcara como `rejected`.

Comprar participacion

Selecciona una factura publicada o parcialmente fondeada y ejecuta la compra desde este tab.

POST /facturas/{id}/comprar

Consultar tracking

Revisa el historial de eventos de una factura registrada en el POC.

GET /facturas/{id}/tracking

Escribe un `invoice_id` o usa el boton de una factura en el tab de compras.

Estado del servicio

Verificacion minima del backend y actividad reciente.

GET /health

Guia de uso

1. En Path 1 registra una factura.
2. Si queda `published`, ve a Path 2.
3. Compra una parte o el total.
4. Revisa eventos en Tracking.
